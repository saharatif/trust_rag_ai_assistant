## Endpoints

```http
GET  /health
POST /ingest
```

## Acceptance Criteria

- [x] App runs locally with `uvicorn`
- [x] `GET /health` returns `{"status": "ok", "service": "TrustRAG API"}` with HTTP 200
- [x] `POST /ingest` accepts valid sample documents
- [x] Empty or invalid documents are rejected with a clean error
- [x] Documents are split into chunks with metadata preserved
- [x] Chunking behavior is covered by automated tests
- [x] README includes local setup and run instructions
- [x] Health and ingestion endpoints have basic per-client rate limits

---

## Key Design Notes

- Use Pydantic models for all request/response validation
- Chunk size and overlap must be configurable via environment variables
- Health and ingestion rate limits are configurable via environment variables
- All errors must be logged, never silently swallowed
- No secrets committed — all config via `.env`
