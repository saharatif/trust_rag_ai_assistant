## CI/CD Pipeline Steps

```text
1. Checkout code
2. Set up Python 3.11
3. Install dependencies (requirements.txt)
4. Run pytest
5. Build Docker image

Optional (if AWS credentials configured):
6. Authenticate with AWS
7. Push image to ECR
8. Deploy CloudFormation stack
```

## AWS Architecture

```text
GitHub Actions
    ↓
Docker Build
    ↓
Push to AWS ECR
    ↓
CloudFormation Deploy
    ↓
ECS Fargate Service
    ↓
Application Load Balancer / API Gateway
    ↓
CloudWatch Logs
```

## CloudFormation Components

- ECR repository
- ECS cluster + task definition + service
- IAM execution role
- CloudWatch log group
- Security group
- Load balancer or API Gateway placeholder
- Environment variable configuration

## Acceptance Criteria

- [ ] `docker-compose up` starts the full application
- [ ] `GET /health` works inside the Docker container
- [ ] All tests pass locally with `pytest`
- [ ] CI pipeline runs and passes on push/PR
- [ ] CI pipeline builds the Docker image successfully
- [ ] Failed tests fail the pipeline
- [ ] CloudFormation template is valid and includes all required components
- [ ] `docs/AWS_DEPLOYMENT.md` documents the full deployment process
- [ ] End-to-end demo can be completed following `docs/DEMO_SCRIPT.md`

---

## Key Design Notes

- No secrets in `Dockerfile` or `docker-compose.yml` — use env vars only
- Docker image must be built from a non-root user for security
- CI must fail fast on test failures before building the image
- CloudFormation template must be deployable without manual console steps
