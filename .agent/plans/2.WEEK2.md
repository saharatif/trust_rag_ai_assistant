## Endpoints

```http
POST /retrieve
POST /chat
```

## Example `/chat` Response

```json
{
  "answer": "Employees can claim travel reimbursement when...",
  "sources": [
    {
      "document_id": "policy_001",
      "title": "Travel Policy",
      "chunk_id": "policy_001_chunk_002"
    }
  ],
  "confidence": "medium",
  "status": "answered"
}
```

## Acceptance Criteria

- [ ] Documents can be ingested and embedded
- [ ] `POST /retrieve` returns top-k relevant chunks with similarity scores
- [ ] `POST /chat` returns a grounded answer with source citations
- [ ] Unsupported questions return a safe fallback (not a hallucinated answer)
- [ ] LLM and embedding failures return clean errors and are logged
- [ ] Evaluation script runs against at least 10 test questions
- [ ] Evaluation results are saved to file
- [ ] README explains how to run the full RAG flow

---

## Key Design Notes

- Prompt must instruct the LLM to refuse answering without supporting context
- Retrieval score threshold should be configurable
- Embedding provider failures must not crash the app
- Log every chat request and response for traceability
