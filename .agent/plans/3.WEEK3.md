## Endpoints

```http
POST /chat                        (now runs through agent graph)
POST /review/{answer_id}/approve
POST /review/{answer_id}/reject
POST /review/{answer_id}/edit
```

## Agent Workflow

```text
Receive Question
    ↓
Retrieve Context
    ↓
Calculate Trust Score
    ↓
Draft Answer
    ↓
Evaluate Confidence
    ↓
Conditional Route
    ├── High confidence + high trust → Final Answer
    └── Low confidence or low trust → Human Review
```

## Trust Score Formula

```text
trust_score =
  metadata_score    * 0.30
+ freshness_score   * 0.30
+ source_quality    * 0.20
+ eval_score        * 0.20
```

## Database Tables

```text
documents, chunks, queries, answers,
retrieval_logs, approval_logs, eval_results
```

## Acceptance Criteria

- [ ] Agentic workflow runs end-to-end via LangGraph
- [ ] Trust score is calculated per retrieved chunk
- [ ] Trust score is explainable and appears in logs/response
- [ ] High-confidence + high-trust answers are returned directly
- [ ] Low-confidence or low-trust answers are routed to pending review
- [ ] Approve, reject, and edit review flows work
- [ ] All review decisions are logged in `approval_logs`
- [ ] Audit records (queries, answers, retrieval, scores) are stored in DB
- [ ] Architecture document explains the full workflow

---

## Key Design Notes

- Workflow state must be serializable — store in DB, not in memory
- Conditional routing thresholds must be configurable
- Never finalize a low-trust answer automatically
- Review endpoints must validate `answer_id` exists and is in pending state
